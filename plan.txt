### Project Overview

The project is a **Custom Web Application** built using **Vue.js 3** and **Bootstrap**. It is divided into three stages that evolve from a basic setup to a complex, real-world application. For the **Distinction/High Distinction** path, you must demonstrate "Technique Complexity" (like Vuex and Lazy-loading) and provide a deep-dive video presentation.

---

### 1. Unified Architecture Design

For an HD-standard project, you must move away from simple component-based data fetching and use a **Centralized State Architecture**.

**The Flow:**

1. **UI Layer:** Components (Login, Dashboard) never talk to the API directly.
2. **Logic Layer (Vuex):** The Store handles all business logic, authentication tokens, and "Lazy-loading" counters.
3. **Data Layer:** An external API provides data in chunks to prevent performance lag.

### 2. Implementation Plan (Coding Phases)

#### **Phase 1: Environment & SPA Basics (Stage 1)**

* Initialize the project with **Vite** and **Vue Router**.
* Set up the **Bootstrap 5 grid system** in `App.vue`.
* Create basic routes: `Home.vue`, `News.vue`, and `About.vue`.

#### **Phase 2: Authentication & CRUD (Stage 2 - Week 10)**

* **Authentication:** Create a Login page. Store the user's status in the Vuex store so the whole app knows if they are logged in.
* **CRUD Operations:** * **Create:** A form to post new entries.
* **Read:** A list view to display data.
* **Update/Delete:** Buttons on each item to edit or remove records via API.



#### **Phase 3: The HD Deep Dive (Stage 3 - Week 11 + Lecturer Tip)**

* **Vuex Migration:** Move all arrays and objects into the Vuex `state`.
* **Lazy-Loading Implementation:** Solve the "large API response" problem.
* Code a function that fetches only 10 items at a time.
* When the user clicks "Load More," fetch the next 10 and **append** them to the Vuex state.

### 3. The Ultimate Technical Checklist

**Framework & Styling Requirements:**

* [ ] Must use **Vue.js version 3**.
* [ ] Must use **Bootstrap Grid System** (Rows and Columns).
* [ ] Must use build tools (**Vite** or **Vue CLI**).
* [ ] Deployment must be on the **Mercury Server**.

**Functionality Requirements (Week 10):**

* [ ] **Login:** POST request to verify user.
* [ ] **Route Guards:** Redirect unauthenticated users to Login.
* [ ] **Full CRUD:** GET (Read), POST (Create), PUT (Update), and DELETE methods implemented.
* [ ] **External API:** Must fetch data from a real web service.

**Advanced Requirements (Week 11 & HD Criteria):**

* [ ] **Vuex Store:** Use `state`, `mutations`, and `actions`.
* [ ] **Async Logic:** All `fetch()` calls must happen inside Vuex Actions.
* [ ] **Lazy-Loading:** Implement pagination to handle large datasets (as suggested by the lecturer).
* [ ] **Reactive UI:** The UI must update automatically when the Vuex state changes.
